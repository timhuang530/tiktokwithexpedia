# V1.3 Iteration Notes

Date: `2026-05-29`
Version: `V1.3`
Focus: `Commerce message fidelity pass`

## Summary

This iteration focuses on improving the new Expedia commerce message inside chat so the hotel card and restaurant card feel closer to the supplied Instagram-style reference. The work stays inside the current chat shell and does not introduce a new interaction model.

## Files Changed

### `src/App.tsx`

- Added and refined the `commerce-group` message flow so Expedia can send one hotel card and one restaurant card after the fixed itinerary sequence.
- Expanded the commerce item data model to support richer fields such as `location`, `feature`, `ratingLabel`, and `note`.
- Replaced earlier first-pass commerce content with the latest content direction:
  - Hotel: `Mauna Lani, Auberge Collection`
  - Restaurant: `Dining at Hard Rock Cafe Honolulu`
- Updated the hotel detail and restaurant shelf seed data so click-through screens remain consistent with the in-chat cards.
- Reordered card metadata so the rating module appears in a consistent position and secondary content sits below it.

Why:

- The first implementation proved the flow but looked too much like a prototype.
- The later user feedback prioritized visual fidelity, cleaner hierarchy, and data that matches the provided references.
- Keeping detail screens in sync prevents the user from seeing one hotel / restaurant in chat and another one after tapping.

### `src/App.css`

- Rebuilt the commerce card visual structure around a more reference-like layout:
  - larger photo-first cards
  - tighter message header
  - clearer rating module
  - stronger CTA button styling
- Added styles for:
  - fixed media height
  - fixed title / rating / metadata sections
  - aligned CTA placement across both cards
  - green rating badge with stacked review copy
  - metadata stack below the rating block
- Tuned spacing so the two cards now align vertically even when the hotel and restaurant have different metadata lengths.

Why:

- The earlier card version was too tall, too text-heavy, and visually inconsistent with the supplied reference.
- The user explicitly asked for shorter cards, wider cards, real photos, clearer rating information, more button-like CTAs, and matching vertical alignment.
- Fixed section heights make both cards feel like one system instead of two unrelated blocks.

### `public/media/oahu-mauna-lani-hotel-photo.jpg`

- Added a new local hotel image for the current hotel card.

Why:

- The previous card used an earlier placeholder direction.
- Local assets are more stable for demos than relying on remote images at runtime.

### `public/media/oahu-hard-rock-honolulu-photo.jpg`

- Added a new local restaurant image for the current restaurant card.

Why:

- The new restaurant content direction changed from the earlier Waikiki dining placeholder to `Hard Rock Cafe Honolulu`.
- A local image keeps rendering predictable in demo environments.

Why:
